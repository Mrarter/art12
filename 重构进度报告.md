## 🎨 价格增长机制

### 功能说明

作品价格会根据以下因素动态增长：

| 因素 | 影响 | 说明 |
|------|------|------|
| 发布时间 | 每日增长 0.02% | 发布30天后提升至 0.03%/天 |
| 艺术家知名度 | 1x - 3x | 大师级 3x、人气艺术家 2x、认证艺术家 1.5x |
| 浏览量 | 1x - 1.5x | 100+/500+/1000+/5000+ 分别提升 |
| 收藏量 | 1x - 1.5x | 5+/20+/50+/100+ 分别提升 |
| 销售次数 | 每次 +5% | 每售出一次价格上涨5% |

**最大涨幅限制**：原价的 5 倍

### 核心代码

```
PriceGrowthService.java
├── calculatePriceRise()     # 计算综合增长率
├── calculateCurrentPrice()  # 计算实时价格
├── calculateTimeMultiplier()    # 时间因素
├── calculateBadgeMultiplier()   # 艺术家知名度
├── calculateViewMultiplier()    # 浏览量
├── calculateFavoriteMultiplier() # 收藏量
└── updateAllPriceRise()     # 定时任务更新价格
```

### 数据库变更

执行 `backend/sql/price_growth_update.sql` 添加以下字段：

```sql
ALTER TABLE artwork ADD COLUMN price_rise DECIMAL(10,4) DEFAULT 0;
ALTER TABLE artwork ADD COLUMN author_badge VARCHAR(50);
ALTER TABLE artwork ADD COLUMN author_avatar VARCHAR(512);
ALTER TABLE artwork ADD COLUMN ownership_type INT DEFAULT 1;
ALTER TABLE artwork ADD COLUMN artwork_code VARCHAR(20);
ALTER TABLE artwork ADD COLUMN view_count INT DEFAULT 0;
```

### 定时任务

- 默认每小时执行一次价格更新
- 可调整为每天凌晨2点执行（修改 cron 表达式）

### 核心代码

## 🏷️ 分类权重排序

### 功能说明

分类管理使用**权重排序**替代原有的排序字段：

| 配置项 | 说明 |
|--------|------|
| 权重字段 | `weight`，数值越大排序越靠前 |
| 默认权重 | 新增分类默认权重为 0 |
| 排序方向 | 降序排列（权重大的在前） |

### 核心变更

1. **数据库字段**：`sort` → `weight`
2. **实体类**：`Category.java` 新增 `@TableField("weight")` 映射
3. **Service排序**：`orderByAsc` → `orderByDesc`
4. **前端表单**：标签名改为"权重"，添加说明提示

### 数据库升级

执行 `backend/sql/category_weight_update.sql`：

```sql
ALTER TABLE `artwork_category` 
CHANGE COLUMN `sort` `weight` INT DEFAULT 0 COMMENT '权重，数值越大排序越靠前';

UPDATE `artwork_category` SET `weight` = `weight` * 10;  -- 可选：放大权重值
```

## 👨‍🎨 艺术家认证管理

### 功能说明

艺术家认证管理页面已全面升级，支持完整的认证流程：

| 功能 | 说明 |
|------|------|
| Tab 切换 | 待审核 / 已认证 / 未通过 / 全部 |
| 状态计数 | 各状态显示待处理数量 badge |
| 添加艺术家 | 后台手动添加艺术家并设置认证等级 |

| 等级设置 | 设置艺术家认证等级（大师级/人气/认证/普通） |
| 取消认证 | 取消已认证艺术家的资格 |
| 重新认证 | 允许被拒绝的艺术家重新申请 |

### 认证等级

| 等级 | badge 值 | 说明 |

```
/admin/user/artist/approve  # 通过认证
/admin/user/artist/reject   # 拒绝认证
/admin/user/artist/revoke   # 取消认证
/admin/user/artist/badge    # 设置等级
/admin/user/artist/{id}     # 艺术家详情
```

### 数据库表

## 💼 艺荐官管理

### 功能说明

艺荐官（推广员）管理页面支持完整的推广员管理：

| 功能 | 说明 |
|------|------|
| 添加艺荐官 | 后台手动添加用户为艺荐官，设置等级 |
| 艺荐官列表 | 显示所有艺荐官，支持按用户ID、等级筛选 |
| 状态统计 | 显示总数、本月新增、待结算佣金 |
| 团队查看 | 查看艺荐官的下级团队成员 |
| 佣金查看 | 查看艺荐官的佣金记录 |
| 冻结/解冻 | 控制艺荐官账户状态 |

### 艺荐官等级

| 等级 | 值 | 说明 |
|------|-----|------|
| 普通艺荐官 | 1 | 初级推广员 |
| 高级艺荐官 | 2 | 中级推广员 |
| 金牌艺荐官 | 3 | 高级推广员 |

### 核心接口

```
/admin/user/promoter/list      # 艺荐官列表
/admin/user/promoter/add       # 添加艺荐官
/admin/user/promoter/freeze    # 冻结/解冻艺荐官
/admin/user/promoter/{userId}  # 艺荐官详情
/admin/user/promoter/team/{userId}    # 团队成员
/admin/user/promoter/commission/{userId}  # 佣金记录
```

### 佣金计算规则

- 直接推荐佣金：一级下线购买作品的 10-15%
- 团队佣金：二级及以下下线购买作品的一定比例
