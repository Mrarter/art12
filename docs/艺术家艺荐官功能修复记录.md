## 一、添加艺术家功能问题

### 问题1：user_account 表缺少 user_uid 字段

**错误信息**:
```
Unknown column 'user_uid' in 'field list'
```

**原因**: 代码尝试向 `user_account` 表插入 `user_uid` 字段，但该字段在数据库中不存在

**修复方法**:
```sql
-- 为 user_account 表添加 user_uid 字段
ALTER TABLE user_account ADD COLUMN user_uid VARCHAR(19) DEFAULT NULL COMMENT '用户UID(新)' AFTER uid;
```

**涉及文件**: `backend/shiyiju-admin/src/main/java/com/shiyiju/admin/service/UserAdminPersistenceService.java`

### 问题2：user_account 表插入语句参数不匹配

**错误信息**:
```
Column count doesn't match value count at row 1
No value specified for parameter 7/8
```

**原因**: SQL INSERT 语句中列数和参数数量不一致

**修复方法**:
修改 `UserAdminPersistenceService.java` 中的 `user_account` 表插入语句：

```java
// 修复前（错误）
INSERT INTO user_account (nickname, mobile, %s, status, register_source, created_at, updated_at, user_uid)
VALUES (?, ?, ?, 'ENABLED', 'ADMIN', ?, ?, ?, ?)  // 10个占位符

// 修复后（正确）
INSERT INTO user_account (nickname, mobile, %s, status, register_source, created_at, updated_at, user_uid, user_no)
VALUES (?, ?, ?, 'ENABLED', 'ADMIN', ?, ?, ?, ?)  // 8个占位符
```

**注意**: 同时需要提供 `user_no` 字段（必填），使用 `U + 时间戳` 格式生成

## 四、经验总结

1. **动态表结构处理**: 系统支持多表切换（user_account/users/sys_user），不同表的字段可能不同
2. **必填字段检查**: 插入数据前需确保所有 NOT NULL 字段都有值
3. **字段存在性检查**: 使用 `schemaInspector.hasColumn()` 检查字段是否存在后再插入
4. **UID 格式**: 19位格式 `USR + YYYYMMDD + 4位序列号 + 4位随机码`
