## 🚀 初始化步骤

### 方法一：Docker环境自动初始化

如果使用Docker部署，MySQL容器会自动初始化基础数据：

```bash
cd deploy/docker
docker-compose up -d mysql
# 等待MySQL启动完成
docker-compose up -d
```

### 方法二：手动初始化

#### 步骤1：创建数据库

```sql
CREATE DATABASE IF NOT EXISTS shiyiju 
CHARACTER SET utf8mb4 
COLLATE utf8mb4_unicode_ci;
```

#### 步骤2：导入基础表结构

```bash
# 导入基础表
mysql -u root -p shiyiju < documents/db/shiyiju_user.sql
mysql -u root -p shiyiju < documents/db/shiyiju_product.sql
mysql -u root -p shiyiju < documents/db/shiyiju_order.sql
mysql -u root -p shiyiju < documents/db/shiyiju_auction.sql
mysql -u root -p shiyiju < documents/db/shiyiju_promotion.sql
mysql -u root -p shiyiju < documents/db/shiyiju_community.sql
mysql -u root -p shiyiju < documents/db/shiyiju_message.sql
```

#### 步骤3：导入新增功能表

```bash
# 文件服务表
mysql -u root -p shiyiju < backend/shiyiju-file/sql/file_info.sql

# 评价系统表
mysql -u root -p shiyiju < backend/shiyiju-product/sql/review.sql

# 物流追踪表
mysql -u root -p shiyiju < backend/shiyiju-order/sql/logistics.sql
```


## 📊 新增表结构说明

### 1. file_info - 文件信息表

存储上传文件的基本信息：

```sql
CREATE TABLE file_info (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  file_name VARCHAR(255) NOT NULL COMMENT '文件名',
  original_name VARCHAR(255) COMMENT '原始文件名',
  file_path VARCHAR(500) NOT NULL COMMENT '文件路径',
  file_url VARCHAR(500) COMMENT '访问URL',
  file_size BIGINT COMMENT '文件大小(字节)',
  file_type VARCHAR(50) COMMENT '文件类型',
  content_type VARCHAR(100) COMMENT 'MIME类型',
  user_id BIGINT COMMENT '上传用户ID',
  business_type VARCHAR(50) COMMENT '业务类型',
  business_id BIGINT COMMENT '业务ID',
  status TINYINT DEFAULT 1 COMMENT '状态:0-禁用,1-正常',
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_user_id (user_id),
  INDEX idx_business (business_type, business_id),
  INDEX idx_create_time (create_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='文件信息表';
```

### 2. review - 作品评价表

存储用户对购买作品的评价：

```sql
CREATE TABLE review (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  order_id BIGINT NOT NULL COMMENT '订单ID',
  order_item_id BIGINT COMMENT '订单项ID',
  artwork_id BIGINT NOT NULL COMMENT '作品ID',
  user_id BIGINT NOT NULL COMMENT '用户ID',
  user_nickname VARCHAR(50) COMMENT '用户昵称',
  user_avatar VARCHAR(255) COMMENT '用户头像',
  rating TINYINT NOT NULL COMMENT '总体评分(1-5)',
  quality_rating TINYINT COMMENT '质量评分',
  logistics_rating TINYINT COMMENT '物流评分',
  service_rating TINYINT COMMENT '服务评分',
  content TEXT COMMENT '评价内容',
  images TEXT COMMENT '评价图片(JSON数组)',
  is_anonymous TINYINT DEFAULT 0 COMMENT '是否匿名:0-否,1-是',
  like_count INT DEFAULT 0 COMMENT '点赞数',
  reply_count INT DEFAULT 0 COMMENT '回复数',
  status TINYINT DEFAULT 1 COMMENT '状态:0-隐藏,1-显示',
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_artwork_id (artwork_id),
  INDEX idx_user_id (user_id),
  INDEX idx_order_id (order_id),
  INDEX idx_status (status),
  INDEX idx_create_time (create_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='作品评价表';

CREATE TABLE review_reply (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  review_id BIGINT NOT NULL COMMENT '评价ID',
  user_id BIGINT NOT NULL COMMENT '回复用户ID',
  user_nickname VARCHAR(50) COMMENT '回复用户昵称',
  content TEXT NOT NULL COMMENT '回复内容',
  reply_type VARCHAR(20) DEFAULT 'seller' COMMENT '回复类型:seller-商家,user-用户',
  status TINYINT DEFAULT 1 COMMENT '状态:0-隐藏,1-显示',
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_review_id (review_id),
  INDEX idx_user_id (user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='评价回复表';
```

### 3. logistics - 物流信息表

存储订单物流信息：

```sql
CREATE TABLE logistics (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  order_id BIGINT NOT NULL COMMENT '订单ID',
  company_code VARCHAR(50) COMMENT '物流公司编码',
  company_name VARCHAR(100) COMMENT '物流公司名称',
  tracking_no VARCHAR(100) NOT NULL COMMENT '运单号',
  status TINYINT DEFAULT 0 COMMENT '状态:0-暂无,1-已发货,2-运输中,3-派送中,4-已签收,5-拒收,6-退件',
  ship_time DATETIME COMMENT '发货时间',
  receive_time DATETIME COMMENT '签收时间',
  receiver_name VARCHAR(50) COMMENT '收货人姓名',
  receiver_phone VARCHAR(20) COMMENT '收货人电话',
  receiver_address VARCHAR(255) COMMENT '收货地址',
  remark TEXT COMMENT '备注',
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_order_id (order_id),
  INDEX idx_tracking_no (tracking_no),
  INDEX idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='物流信息表';

CREATE TABLE logistics_track (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  logistics_id BIGINT NOT NULL COMMENT '物流ID',
  tracking_no VARCHAR(100) NOT NULL COMMENT '运单号',
  status VARCHAR(50) COMMENT '状态',
  description TEXT COMMENT '描述',
  location VARCHAR(100) COMMENT '地点',
  track_time DATETIME COMMENT '轨迹时间',
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_logistics_id (logistics_id),
  INDEX idx_tracking_no (tracking_no),
  INDEX idx_track_time (track_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='物流轨迹表';
```

## ⚙️ 配置文件更新

### 数据库连接配置

在 `application.yml` 中配置数据库连接：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/shiyiju?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    username: shiyiju
    password: your_password
    driver-class-name: com.mysql.cj.jdbc.Driver
```

### 文件存储配置

```yaml
upload:
  storage-type: local  # local-本地存储, cos-腾讯云COS, oss-阿里云OSS
  local:
    path: /var/shiyiju/uploads
  cdn-url: https://cdn.yourdomain.com

# 腾讯云COS配置(可选)
tencent:
  cos:
    bucket: shiyiju-uploads
    region: ap-guangzhou
    secret-id: your_secret_id
    secret-key: your_secret_key

# 阿里云OSS配置(可选)
aliyun:
  oss:
    bucket: shiyiju-uploads
    endpoint: oss-cn-shenzhen.aliyuncs.com
    access-key-id: your_access_key_id
    access-key-secret: your_access_key_secret
```

## 🐛 常见问题

### Q1: 导入SQL时报错"Table already exists"

**解决方案**：先删除旧表再导入
```sql
DROP TABLE IF EXISTS table_name;
```

### Q2: 字符集不正确，中文显示乱码

**解决方案**：确保数据库和表的字符集都是 utf8mb4
```sql
ALTER DATABASE shiyiju CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### Q3: 外键约束失败

**解决方案**：按正确顺序导入表，或暂时禁用外键检查
```sql
SET FOREIGN_KEY_CHECKS = 0;
-- 执行导入
SET FOREIGN_KEY_CHECKS = 1;
```

### Q4: Docker环境下数据库初始化失败

**解决方案**：等待MySQL容器完全启动后再导入
```bash
docker-compose up -d mysql
# 等待10秒
docker exec -i shiyiju-mysql mysql -uroot -p123456 shiyiju < backend/shiyiju-file/sql/file_info.sql
```
