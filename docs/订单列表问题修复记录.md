# 订单列表问题修复记录

**日期**: 2026-04-27
**问题类型**: 数据库表结构与代码不一致导致查询失败

---

## 问题：订单列表提示"系统繁忙"

### 错误信息
```
Table 'shiyiju.shiyiju_user' doesn't exist
Unknown column 'cover_image' in 'field list'
```

### 原因分析
代码中使用的表名和字段名与数据库实际结构不匹配

### 修复方法
修改 `backend/shiyiju-admin/src/main/java/com/shiyiju/admin/service/OrderService.java`

#### 1. 表名修正：shiyiju_user → user_account

```java
// 修复前
LEFT JOIN shiyiju_user u ON o.buyer_user_id = u.id

// 修复后
LEFT JOIN user_account u ON o.buyer_user_id = u.id
```

#### 2. 移除不存在的字段：phone

```java
// 修复前：SELECT ... u.phone as buyer_phone ...
// 修复后：移除 phone 字段（user_account 表无此字段）
```

#### 3. 字段名修正：cover_image → cover_url

```java
// 修复前
"SELECT cover_image, artwork_title FROM trade_order_item WHERE order_id = ?"
item.put("cover", items.get(0).get("cover_image"));
item.put("artworkTitle", items.get(0).get("artwork_title"));

// 修复后
"SELECT cover_url, item_title FROM trade_order_item WHERE order_id = ?"
item.put("cover", items.get(0).get("cover_url"));
item.put("artworkTitle", items.get(0).get("item_title"));
```

#### 4. 简化订单商品查询

```java
// 修复前
SELECT toi.*, a.title as artwork_title, a.cover_image 
FROM trade_order_item toi
LEFT JOIN artwork a ON toi.artwork_id = a.id

// 修复后
SELECT toi.item_title, toi.cover_url, toi.unit_price, toi.quantity, toi.subtotal_amount,
       a.title as artwork_title, a.cover_image 
FROM trade_order_item toi
LEFT JOIN artwork a ON toi.artwork_id = a.id
```

## 经验总结

1. **表结构需与代码严格匹配**: 不同表的字段名可能不同
2. **动态表结构处理**: 系统可能存在多个用户表变体（users, user_account, sys_user）
3. **字段存在性检查**: 使用字段前需确认数据库表结构
4. **UID 格式**: 用户UID字段在 `user_account` 表中为 `uid`
