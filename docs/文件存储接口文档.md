## 📁 存储配置

### 1. 本地存储配置

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 存储类型 | local | 本地存储模式 |
| 本地路径 | `/Users/master/CodeBuddy/art12/uploads` | 文件物理存储路径 |
| CDN地址 | `http://localhost:8087` | 文件访问地址 |
| 图片大小限制 | 10MB | 单张图片最大 |
| 视频大小限制 | 100MB | 单个视频最大 |

### 2. 存储路径规则

```
{uploads_path}/images/{userId}/{yyyy/MM/dd}/{uuid}.{ext}
{uploads_path}/videos/{userId}/{yyyy/MM/dd}/{uuid}.{ext}
```

**示例**：
```
/Users/master/CodeBuddy/art12/uploads/images/5/2026/04/25/abc123def456.jpg
```

## 🗄️ 数据库表结构

### 1. 文件信息表 (file_info)

```sql
CREATE TABLE file_info (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    file_name VARCHAR(255) NOT NULL COMMENT '文件名',
    original_name VARCHAR(255) COMMENT '原始文件名',
    file_path VARCHAR(500) NOT NULL COMMENT '文件路径',
    file_url VARCHAR(500) NOT NULL COMMENT '文件URL',
    file_size BIGINT COMMENT '文件大小(字节)',
    file_type VARCHAR(50) NOT NULL COMMENT '文件类型(image/video)',
    mime_type VARCHAR(100) COMMENT 'MIME类型',
    extension VARCHAR(20) COMMENT '文件扩展名',
    user_id BIGINT COMMENT '上传用户ID',
    biz_type VARCHAR(50) COMMENT '业务类型(artwork/avatar/post等)',
    biz_id BIGINT COMMENT '关联业务ID',
    status TINYINT DEFAULT 1 COMMENT '状态(0-禁用,1-正常)',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_user_id (user_id),
    INDEX idx_file_type (file_type),
    INDEX idx_biz_type_id (biz_type, biz_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='文件信息表';
```

### 2. 作品表 (artwork) - 图片字段

```sql
-- 作品封面图
cover_image VARCHAR(512) NOT NULL COMMENT '封面图',

-- 作品图片列表 (JSON格式)
images JSON DEFAULT NULL COMMENT '图片列表JSON',
```

### 3. 用户表 (sys_user) - 头像字段

```sql
-- 用户头像
avatar VARCHAR(512) DEFAULT NULL COMMENT '头像URL',
```

### 4. Banner表 (banner) - 图片字段

```sql
-- Banner图片
image_url VARCHAR(512) NOT NULL COMMENT '图片URL',
```

### 5. 帖子表 (community_post) - 图片字段

```sql
-- 帖子图片列表
images JSON DEFAULT NULL COMMENT '图片列表',

-- 视频URL
video_url VARCHAR(512) DEFAULT NULL COMMENT '视频URL',
```

## 📡 文件服务接口

### Base URL
```
http://localhost:8087/upload
```

```http

Content-Type: multipart/form-data
X-User-Id: {userId}
```

**请求参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| file | File | 是 | 图片文件 |

## 🛠️ 前端调用示例

### 1. Element Plus 上传组件

```vue
<template>
  <el-upload
    class="avatar-uploader"
    :show-file-list="false"
    :before-upload="beforeUpload"
    :http-request="handleUpload"
    action="#"
  >
    <img v-if="imageUrl" :src="imageUrl" class="avatar" />
    <el-icon v-else class="avatar-uploader-icon"><Plus /></el-icon>
  </el-upload>
</template>

<script setup>
import request from '@/api/request'

const imageUrl = ref('')

const beforeUpload = (file) => {
  const isImage = file.type === 'image/jpeg' || file.type === 'image/png'
  const isLt5M = file.size / 1024 / 1024 < 5
  if (!isImage) {
    ElMessage.error('只能上传 JPG/PNG 图片')
    return false
  }
  if (!isLt5M) {
    ElMessage.error('图片大小不能超过 5MB')
    return false
  }
  return true
}

const handleUpload = async (options) => {
  const { file } = options
  const formData = new FormData()
  formData.append('file', file)
  
  try {
    const res = await request.post('/upload/image', formData, {
      headers: { 'Content-Type': 'multipart/form-data' }
    })
    imageUrl.value = res.url
    ElMessage.success('上传成功')
  } catch (e) {
    ElMessage.error('上传失败')
  }
}
</script>
```

### 2. 批量上传

```javascript
const handleBatchUpload = async (fileList) => {
  const formData = new FormData()
  fileList.forEach(file => {
    formData.append('files', file)
  })
  
  const res = await request.post('/upload/images', formData, {
    headers: { 'Content-Type': 'multipart/form-data' }
  })
  
  // 返回URL数组
  return res.map(item => item.url)
}
```

## 📝 SQL查询示例

### 查询作品图片

```sql
-- 查看作品封面图
SELECT id, title, cover_image, images FROM artwork WHERE cover_image IS NOT NULL;

-- 查看某作品的所有图片
SELECT * FROM file_info WHERE biz_type = 'artwork' AND biz_id = 1;
```

### 查询用户上传文件

```sql
-- 查看某用户上传的所有文件
SELECT * FROM file_info WHERE user_id = 5 ORDER BY create_time DESC;

-- 统计用户上传文件数量
SELECT user_id, COUNT(*) as file_count FROM file_info GROUP BY user_id;
```

### 查询Banner图片

```sql
-- 查看所有Banner
SELECT id, title, image_url FROM banner WHERE status = 1;
```

```yaml

  port: 8087

spring:
  servlet:
    multipart:
      max-file-size: 50MB
      max-request-size: 100MB

  storage-type: local          # 存储类型: local/cos
```

## 🚀 常见问题

### Q1: 上传失败，提示"文件不能为空"

**原因**：文件未选择或文件为空
**解决**：确保选择了有效的文件

### Q2: 上传失败，提示"文件大小超出限制"

**原因**：文件超过限制大小
**解决**：检查 `upload.max-image-size` 配置，当前限制为10MB

### Q3: 图片上传成功但无法访问

**原因**：CDN地址配置错误或文件路径不存在
**解决**：
1. 检查CDN地址是否正确
2. 确认文件已保存到指定路径

### Q4: 如何切换到云存储(COS)?

**步骤**：
1. 修改 `upload.storage-type` 为 `cos`
2. 配置腾讯云COS的 `secret-id`、`secret-key`、`bucket`
3. 重新部署服务
