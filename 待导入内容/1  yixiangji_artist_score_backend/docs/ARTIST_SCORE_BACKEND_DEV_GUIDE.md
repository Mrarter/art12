# 艺术家评分与价格上涨后端开发说明 V1.0

## 一、业务定位

该模块用于艺术品交易平台的核心交易机制：

- 艺术家越有资质、越活跃、越能成交，评分越高。
- 艺术家评分影响作品价格上涨速度。
- 作品成交、收藏行为会触发价格变化。
- 所有价格变化必须记录日志，方便后台审计。

## 二、评分结构

|---|---:|

| 活跃度 | 100 |
| 作品质量 | 150 |
| 藏家评价 | 100 |
| 学术资质 | 100 |
| 互联网资质 | 50 |

## 三、学术资质分

封顶 100 分。

- 中央美术学院 / 中国美术学院 / 清华美院：40 分
- 其他美术学院：30 分
- 普通艺术院校：15 分
- 硕士 / 博士：10 分
- 教授：40 分
- 副教授：30 分
- 讲师：20 分
- 美协身份：20 分

## 四、互联网资质分

封顶 50 分。

粉丝分：

- 1万-5万：5 分
- 5万-10万：10 分
- 10万-50万：15 分
- 50万-100万：20 分
- 100万以上：25 分

内容质量分：最高 15 分。

转化能力分：最高 10 分。

## 六、价格上涨规则

### 每日上涨

每日上涨率根据艺术家总评分决定：

| 等级/分数 | 日涨幅 |
|---|---:|
| 900+ | 0.5% |
| 750+ | 0.4% |
| 500+ | 0.3% |
| 250+ | 0.2% |
| 250以下 | 0.1% |

### 成交触发

每成交一次，价格上涨 2%。

### 收藏触发

每触发一次收藏涨价，价格上涨 0.5%。

实际生产中建议增加：每新增 10 个收藏触发一次，避免每次收藏都涨价。

## 七、Codex 后续开发任务

### 必须补齐

1. MyBatis XML：findByArtistId、upsert、findById、updateCurrentPrice、insert。
2. 统一返回结构：ApiResponse<T>。
3. 统一异常处理。
4. 定时任务：每天凌晨自动更新所有在售作品价格。
5. 后台审核接口：艺术家身份资质审核。
6. 后台人工调价接口。
7. 单元测试和接口测试。

### 可选增强

1. 引入 Redis 缓存艺术家评分。
2. 价格上涨增加冷却机制：7 天无成交停止上涨，14 天无成交回调。
3. 评分计算接入订单表、收藏表、浏览表、评价表。
4. 接入社交平台账号验证。
