## 二、Nacos 安装

### 方式1: 使用国内镜像下载 (推荐)

```bash
cd /Users/master/CodeBuddy/art12

# 删除损坏的文件
rm -f nacos-server.tar.gz nacos

# 使用镜像下载
curl -L -o nacos-server.tar.gz https://ghproxy.com/https://github.com/alibaba/nacos/releases/download/2.2.3/nacos-server-2.2.3.tar.gz

# 解压
tar -xzf nacos-server.tar.gz

# 单机模式启动
cd nacos
sh bin/startup.sh -m standalone
```

### 方式2: IDE中配置启动

如果Nacos下载困难，可以配置各服务跳过Nacos依赖：

1. 在各服务的 `application.yml` 中注释掉:
```yaml
# spring:
#   cloud:
#     nacos:
#       discovery:
#         server-addr: 127.0.0.1:8848
```

2. 将 `spring.cloud.nacos.discovery.enabled` 设置为 `false`

## 三、启动顺序

### 方式A: 使用IDE (推荐)

1. **启动 Nacos** (方式1完成后)
   - 访问 http://localhost:8848/nacos
   - 用户名/密码: nacos/nacos

2. **按顺序启动微服务**:
   ```
   1. shiyiju-gateway (8080)
   2. shiyiju-user (8081)
   3. shiyiju-product (8082)
   4. shiyiju-order (8083)
   5. shiyiju-auction
   6. shiyiju-promotion
   7. shiyiju-community
   8. shiyiju-message
   9. shiyiju-file
   ```

### 方式B: 命令行启动

```bash
# 进入后端目录
cd /Users/master/CodeBuddy/art12/backend

# 启动网关
java -jar shiyiju-gateway/target/shiyiju-gateway-1.0.0.jar &

# 启动用户服务
java -jar shiyiju-user/target/shiyiju-user-1.0.0.jar &

# 启动商品服务
java -jar shiyiju-product/target/shiyiju-product-1.0.0.jar &

# 启动订单服务
java -jar shiyiju-order/target/shiyiju-order-1.0.0.jar &
```

## 七、常见问题

### Q: MySQL连接失败
```bash
# 检查MySQL服务
brew services list | grep mysql

# 重启MySQL
brew services restart mysql@8.0
```

### Q: Redis连接失败
```bash
# 检查Redis服务
brew services list | grep redis

# 重启Redis
brew services restart redis
```

### Q: Nacos启动失败
```bash
# 查看日志
cat nacos/logs/start.out
```

### Q: 服务注册失败
确保Nacos已启动并正常运行后再启动微服务。
